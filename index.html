BbSync.org

Welcome to BbSync.org

BbSync.org is an Apache-licensed, open source project that allows Java developers with no web service programming experience to create functional web service client applications that interact with Blackboard Learn™.

Why BbSync.org?

Blackboard Learn™ ships with a rich set of default web services that allow web service clients to interact with most of Learn's core functionality.  That is, these client web services are essentially a subset of Blackboard Learn's Building Blocks™ functionality that can be run on a remote computer.  This includes the ability to create/update/delete users, courses, and memberships - as well as many other increadibly useful functions.

Unfortunately, many people have found that getting started with these web services is difficult at best. Much of the documentation that exists for Learn's web services is quite dated and the sample code provided with Learn can be extremely difficult to understand.

BbSync.org is more than just code.  It is an attempt to:

  1. Provide easy-to-use, accessible code that is backed by easy-to-understand, functioning tests.
  2. Provide usefull documentation and tutorials for working with Blackboard Learn's web services.
  3. Promote a community of Blackboard Learn web service developers and contributers.

Community is the most important part of this equation.  That's why this project is on GitHub.  Through GitHub, everyone will be able to fork their own copy of this project and contribute back to the original.  If you're not familiar with how to do this, please watch this introductory video on GitHub basics.

How Does it work?

BbSync.org objects implement a uniform web service client object interface in order to simplify how you work with individual Blackboard Learn™ web service objects.

  • persist():Serializable - creates or updates a web service object.
  • retrieve():Object - gets a web service object.
  • delete():boolean - deletes a web service object.

For instance, to create a simple course in Blackboard Learn™ using BbSync.org, all you would need to do is this: 


       BbCourse bb_course = new BbCourse();
       bb_course.setCourseId("tst_prst_crs");
       bb_course.setName("Test Persist Course");
       String id = (String) bb_course.persist();

To get the course in the example above from your Blackboard Learn™ server, do this: 


        BbCourse bb_course = new BbCourse();
        bb_course.setId(id);
        bb_course = (BbCourse) bb_course.retrieve();

Finally, to delete the course in the example above, do this: 


        BbCourse bb_course = new BbCourse();
        bb_course.setId(id);
        bb_course.delete();

Using BbSync.org, the same basic pattern can be applied to all of the Blackboard Learn™ web service client objects such as users, memberships, terms, etc. Plus there's a whole lot more to BbSync.org that should make working with Blackboard Learn™ web services easier than ever before.
